# Проверка адресов на baza-winner.ru

## Установка (один раз)

```bash
pip install playwright pandas openpyxl --break-system-packages
python -m playwright install webkit
```

## Запуск

```bash
python baza_search.py
```

При запуске скрипт сам создаёт рядом с собой папки `input/` и
`output/` (если их ещё нет). Положите все xlsx-файлы со списками
адресов в `input/` — скрипт по очереди обработает каждый файл и для
каждого сохранит отдельный результат в `output/`, например:
`input/лои_июнь.xlsx` -> `output/лои_июнь_results.csv`.

Можно также по-прежнему указать конкретный файл вручную:

```bash
python baza_search.py --input лои_июнь.xlsx --output results.csv
```

При первом запуске откроется окно браузера. Залогиньтесь на сайте
вручную, как обычно, затем вернитесь в терминал и нажмите Enter —
скрипт продолжит работу. Сессия сохранится в папке `.browser_profile`
рядом со скриптом, так что при следующих запусках логиниться заново
не придётся (если только сайт сам не разлогинит сессию по таймауту).

## Формат входного файла

Первый столбец, без заголовка, по одному адресу в строке, например:

```
Беловежская улица, дом 21
Ивантеевская улица, дом 28, корпус 3
```

## Формат результата

Для каждого входного файла создаётся свой CSV в `output/`:

```
Адрес;Найдено вариантов;Ссылка
Беловежская улица, дом 21;10;https://online.baza-winner.ru/...
```

Результаты дописываются в файл по ходу работы — если скрипт упадёт
или вы его остановите на середине, уже обработанные адреса не
потеряются.

## Ollama-режим (автопочинка селекторов)

Сайт периодически меняет вёрстку и подписи, из-за чего «хрупкие»
селекторы (плейсхолдер поля адреса, текст кнопки «Найти», иконка
«поделиться» и т.п.) перестают совпадать и поиск молча ничего не
находит. Чтобы чинить это без правки кода, есть необязательный
Ollama-режим: при провале поиска локальная LLM смотрит на текущую
страницу и записывает исправленные селекторы в `config.json`.

Установка (один раз) — ставит Ollama и модель `qwen2.5:7b-instruct`,
поднимает локальный сервер и включает режим:

```bash
./setup_ollama.sh
```

После этого запускайте парсер как обычно (`./launch.sh` или
`python baza_search.py`). Как только адрес не находится, в консоли
появится строка `[ollama] ...`, и если модель поняла, что поменялось,
она обновит `config.json`. Новые селекторы подхватываются со следующего
адреса — перезапуск не нужен.

Включить режим на один запуск, не трогая конфиг:

```bash
python baza_search.py --ollama
```

Все настраиваемые селекторы и параметры Ollama (`enabled`, `host`,
`model`) лежат в `config.json`. Если файла нет — берутся значения по
умолчанию, а сам режим выключен, и скрипт работает ровно как раньше.

## Важные замечания

- **Селекторы могут потребовать доводки.** Сайт сделан на кастомных
  веб-компонентах (shadow DOM), а не на обычном HTML — структура
  местами хрупкая. Перед прогоном всего списка адресов рекомендую
  проверить скрипт на 2-3 адресах и посмотреть, что происходит в
  открытом окне браузера.
- **Скорость.** В скрипте уже заложены случайные задержки между
  вводом символов (60-220 мс на символ) и между обработкой разных
  адресов (2.5-6.5 секунд). Если сайт всё равно начнёт показывать
  капчу — увеличьте эти интервалы в функциях `type_like_human` и
  `between_requests_delay`.
- **Проверяйте условия использования сайта** на предмет ограничений
  на автоматизированные запросы, чтобы не получить блокировку
  аккаунта.
