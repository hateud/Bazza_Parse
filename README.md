# Проверка адресов на baza-winner.ru

Скрипт по списку адресов из xlsx-файлов ищет объявления на
`w7.baza-winner.ru` и сохраняет по каждому адресу количество найденных
вариантов и ссылку «поделиться» в CSV.

Работает через реальный браузер (Playwright + WebKit): открывает
страницу поиска, вводит улицу и дом «по-человечески» (со случайными
задержками), выбирает подсказки и снимает результат.

---

## Быстрый старт (macOS)

```bash
./launch.sh
```

`launch.sh` сам создаёт виртуальное окружение `.venv`, ставит
зависимости, докачивает браузер WebKit для Playwright и запускает
парсер. Можно передавать те же аргументы, что и самому скрипту:

```bash
./launch.sh --input лои_июнь.xlsx --output results.csv
```

### Ручная установка (если не хотите `launch.sh`)

```bash
pip install playwright pandas openpyxl --break-system-packages
python -m playwright install webkit
python baza_search.py
```

---

## Как пользоваться

1. Положите xlsx-файлы со списками адресов в папку `input/` (она
   создаётся автоматически рядом со скриптом при первом запуске).
2. Запустите скрипт. Он по очереди обработает каждый файл и сохранит
   отдельный результат в `output/`, например:
   `input/лои_июнь.xlsx` → `output/лои_июнь_results.csv`.

Можно обработать один конкретный файл вручную:

```bash
python baza_search.py --input лои_июнь.xlsx --output results.csv
```

### Первый запуск и логин

При первом запуске откроется окно браузера. Залогиньтесь на сайте
вручную, как обычно, затем вернитесь в терминал и нажмите **Enter** —
скрипт продолжит работу.

Сессия сохраняется в папке `.browser_profile` рядом со скриптом, так
что при следующих запусках логиниться заново не придётся (пока сайт сам
не разлогинит сессию по таймауту).

---

## Формат входного файла

Первый столбец, без заголовка, по одному адресу в строке:

```
Беловежская улица, дом 21
Ивантеевская улица, дом 28, корпус 3
```

## Формат результата

Для каждого входного файла создаётся свой CSV в `output/`:

```
Адрес;Найдено вариантов;Ссылка
Беловежская улица, дом 21;10;https://online.baza-winner.ru/...
```

Результаты дописываются в файл по ходу работы — если скрипт упадёт или
вы остановите его на середине, уже обработанные адреса не потеряются.

## Ollama-режим (автопочинка селекторов)

Сайт сделан на кастомных веб-компонентах (shadow DOM) и периодически
меняет вёрстку и подписи. Из-за этого «хрупкие» селекторы (плейсхолдер
поля адреса, текст кнопки «Найти», иконка «поделиться» и т.п.)
перестают совпадать, и поиск молча ничего не находит.

Чтобы чинить это без правки кода, есть **необязательный** Ollama-режим:
при провале поиска локальная LLM смотрит на слепок текущей страницы,
определяет, какой селектор разъехался, и записывает исправление в
`config.json`. Новые селекторы подхватываются со следующего адреса —
перезапуск не нужен.

### Установка (один раз)

```bash
./setup_ollama.sh
```

Скрипт ставит Ollama (через Homebrew или официальный `.zip`), скачивает
модель по умолчанию `qwen3:8b`, поднимает локальный сервер
(`http://localhost:11434`) и включает режим в `config.json`.

### Использование

Дальше запускайте парсер как обычно (`./launch.sh` или
`python baza_search.py`). Когда адрес не находится, в консоли появится
строка `[ollama] ...`; если модель поняла, что поменялось, она обновит
`config.json`.

Включить режим на один запуск, не трогая конфиг:

```bash
python baza_search.py --ollama
```

Если Ollama не установлена или режим выключен — скрипт работает ровно
как раньше, а ошибки автопочинки не роняют прогон (только печатают
строку `[ollama] ...`).

### Какую модель выбрать

Режим не привязан к конкретной модели: задача простая (по слепку
страницы вернуть JSON с исправленными селекторами), поэтому небольшой
инструкт-модели с хорошим structured output достаточно. Сменить модель
можно в `ollama.model` в `config.json` или в переменной `MODEL` в
`setup_ollama.sh`.

| Когда | Модель | ~RAM (Q4) |
|-------|--------|-----------|
| По умолчанию, эта задача | `qwen3:8b` | ~6 ГБ (16 ГБ Mac ок) |
| Хочется точнее | `gemma3:12b`, `mistral-small` | ~10–16 ГБ |
| Сложнее (анализ полного HTML, многошаговость) | `qwen3:14b`, `qwen3:32b` | ~10 / ~20 ГБ (32 ГБ+) |
| Самые тяжёлые задачи | `llama3.3:70b` | ~40 ГБ+ (64 ГБ+) |

- **Русский язык.** Qwen3 и Gemma 3 хорошо работают с русским — это
  важно, потому что подписи на сайте русские.
- **Reasoning-модели** (`deepseek-r1` и дистилляты) с нашим
  `format:json` капризны: они эмитят блок рассуждений `<think>`, и
  чистый JSON приходится выковыривать. Для этого режима лучше обычные
  инструкт-модели (или Qwen3 с выключенным thinking).
- **Актуальный список тегов** — на
  [ollama.com/library](https://ollama.com/library). Модели выходят
  часто, так что таблица выше — отправная точка, а не догма.

### config.json

Все настраиваемые селекторы и параметры Ollama лежат в `config.json`
рядом со скриптом. Файл создаётся автоматически (`setup_ollama.sh` или
первая автопочинка); если его нет, берутся значения по умолчанию из
`ollama_fixer.py`, а режим выключен. Ключевые поля:

- `ollama.enabled` — включён ли режим;
- `ollama.host`, `ollama.model`, `ollama.timeout_seconds` — параметры
  подключения к серверу;
- `selectors.*` — те самые «чинибельные» селекторы (URL поиска, текст
  кнопок, плейсхолдеры и т.п.).

## Важные замечания

- **Селекторы могут потребовать доводки вручную.** Автопочинка чинит
  разъехавшиеся тексты/селекторы, но слом самой логики (порядок шагов,
  структура выпадающих подсказок) по-прежнему требует правки кода.
  Перед прогоном всего списка проверьте скрипт на 2–3 адресах, глядя в
  открытое окно браузера.
- **Скорость и капча.** В скрипте заложены случайные задержки между
  вводом символов (60–220 мс на символ) и между адресами (7–9 секунд).
  Если сайт всё равно начнёт показывать капчу — увеличьте эти интервалы
  в функциях `type_like_human` и `between_requests_delay`.
- **Условия использования сайта.** Проверьте ограничения на
  автоматизированные запросы, чтобы не получить блокировку аккаунта.
